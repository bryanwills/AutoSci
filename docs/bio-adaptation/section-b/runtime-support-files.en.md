## Graph Files

| File | Content | Generated by |
|------|---------|-------------|
| `edges.jsonl` | semantic relationships: paper-paper, paper-concept, claim/experiment/idea/provenance, bio-relation, validation/translation, dataset-version edges <!-- bio-B1/B2/B3 --> | `python3 tools/research_wiki.py add-edge` |
| `citations.jsonl` | bibliographic paper citations (`type: cites`) | `python3 tools/research_wiki.py add-citation` |
| `context_brief.md` | compressed context: claims + gaps + failed ideas + papers + edges (≤8000 chars) | `python3 tools/research_wiki.py rebuild-context-brief` |
| `open_questions.md` | open questions: under-supported claims + open questions from papers/topics | `python3 tools/research_wiki.py rebuild-open-questions` |

Semantic edge format (base):

```json
{"from": "node_id", "to": "node_id", "type": "edge_type",
 "evidence": "...", "confidence": "high|medium|low", "date": "..."}
```

Bio extension — semantic edge format with optional per-edge metadata:

```json
{"from": "node_id", "to": "node_id", "type": "edge_type",
 "evidence": "...", "confidence": "high|medium|low", "date": "...",
 "metadata": { "<edge-type-specific keys>": "<values>" }}
```

Citation format: `{"from": "papers/citing", "to": "papers/cited", "type": "cites", "source": "semantic_scholar|parsed_bib|manual", "date": "..."}`

### Edge type catalogue

Each row lists allowed endpoint types, the required `metadata` keys (if any), and a one-line semantic.
**All non-citation edges require `confidence: high|medium|low`.**

| Edge type | Endpoints | Required `metadata` | Semantic |
|-----------|-----------|---------------------|----------|
| _paper-paper (existing)_ | | | |
| `same_problem_as` | paper ↔ paper (symmetric) | — | tackles the same task |
| `similar_method_to` | paper ↔ paper (symmetric) | — | mechanistically similar method |
| `complementary_to` | paper ↔ paper (symmetric) | — | combinable contributions |
| `builds_on` | paper → paper | — | extends prior work |
| `compares_against` | paper → paper | — | reports a head-to-head |
| `improves_on` | paper → paper | — | beats prior on shared metric |
| `challenges` | paper → paper | — | contests a prior claim |
| `surveys` | paper → paper | — | review-of |
| _paper-concept (existing)_ | | | |
| `introduces_concept` | paper → concept | — | first formal definition |
| `uses_concept` | paper → concept | — | applies without redefining |
| `extends_concept` | paper → concept | — | adds variant / generalisation |
| `critiques_concept` | paper → concept | — | argues against |
| _claim / experiment / provenance (existing)_ | | | |
| `supports`, `contradicts` | paper / experiment → claim | — | evidentiary direction |
| `tested_by` | claim → experiment | — | provenance of evidence |
| `invalidates` | experiment → claim | — | falsification record |
| `addresses_gap` | idea / experiment → claim | — | targets an open question |
| `derived_from`, `inspired_by` | idea → idea / paper | — | provenance of ideation |
| <!-- bio-B1 --> _bio relation (new)_ | | | |
| `targets_protein` | paper / claim / concept → concept (gene-product) | — | drug or modality acts on this protein |
| `binds` | concept → concept (protein-protein, ligand-protein) | — | direct physical interaction |
| `inhibits`, `activates`, `degrades` | paper / claim → concept (protein) | — | functional perturbation direction |
| `phosphorylates`, `ubiquitinates`, `methylates`, `acetylates` | concept (enzyme) → concept (substrate) | — | PTM verb; enzyme-on-substrate |
| `is_substrate_of` | concept (substrate) → concept (enzyme) | — | reverse of the four PTM verbs |
| <!-- bio-B2 --> _validation / translation (new)_ | | | |
| `clinical_trial_for` | claim / paper → concept (drug-or-modality) | `nct_id`, `phase` | a registered trial tests this | 
| `fda_approved_for` | concept (drug) → concept (indication) | `indication`, `year` | regulatory approval record |
| `validates_in_species` | claim / experiment → concept (or wikilink to species name) | `species` | confirmed in this organism |
| <!-- bio-B3 --> _dataset-version provenance (new)_ | | | |
| `dataset_version_used` | experiment → dataset | `version` (and optionally `slug` if it differs from the `to` node id) | pins the dataset snapshot used by the experiment |

**Tooling note**: `tools/research_wiki.py add-edge` should accept `--metadata key=value` repeated; metadata-less edges keep their existing CLI shape. `add-edge` should reject unknown `metadata` keys per edge type.
