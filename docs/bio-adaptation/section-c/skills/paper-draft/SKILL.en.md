# /paper-draft

> Draft a complete LaTeX paper from the PAPER_PLAN.md generated by /paper-plan.
> Write section by section: each section draws material from wiki claims/experiments/papers/concepts,
> generates LaTeX + figures + tables. BibTeX is fetched from DBLP/CrossRef <!-- bio-C10 --> *(or CrossRef/PubMed for bio/clinical)* (following citation-verification).
> Apply de-AI polish after completion (following academic-writing).
> Optional per-section Review LLM review. Output a compilable paper/ directory.
> <!-- bio-C10 --> The skill reads `paper_style` (cs / bio / clinical) from PAPER_PLAN.md metadata; section order, figure conventions, citation style, and Review LLM persona are dispatched accordingly.

## Inputs

- `plan`: path to PAPER_PLAN.md (e.g. `wiki/outputs/paper-plan-sparse-lora-2026-04-08.md`)
- `--review` (optional): enable per-section Review LLM review
- `--sections` (optional): write only specified sections (e.g. `--sections 3,4` writes only Method + Experiments); used for incremental writing

## Outputs

- `paper/` directory (under the wiki project root):
  - `paper/main.tex` — master file (includes each section)
  - `paper/sections/{section-files}.tex` — content per section. <!-- bio-C10 --> *Section file names depend on `paper_style`:*
    - **cs**: `introduction.tex`, `related_work.tex`, `method.tex`, `experiments.tex`, `conclusion.tex`
    - **bio**: `introduction.tex`, `results.tex`, `discussion.tex`, `methods.tex` (note Results before Methods; Methods last)
    - **clinical**: `introduction.tex`, `methods.tex`, `results.tex`, `discussion.tex` (Methods front-loaded)
  - `paper/sections/appendix.tex` (if applicable)
  - `paper/figures/` — generated figures (PDF/PNG)
  - `paper/tables/` — standalone table files (optional)
  - `paper/math_commands.tex` — shared math symbol definitions
  - `paper/references.bib` — verified BibTeX entries
- `wiki/log.md` — appended log entry

## Workflow

**Precondition**: confirm the working directory is the wiki project root (the directory containing `wiki/`, `raw/`, `tools/`).

### Step 1: Initialize Paper Directory

1. Read PAPER_PLAN.md; extract venue, title, section list, **`paper_style`** (from metadata block — error if missing).   <!-- bio-C10 -->
2. If `paper/` directory already exists:
   - Back up as `paper.bak-{timestamp}/`
   - Prompt user to confirm overwrite
3. Create directory structure:
   ```
   paper/
   ├── main.tex
   ├── math_commands.tex
   ├── references.bib
   ├── sections/
   └── figures/
   ```
4. Copy venue template from `templates/` if it exists:
   - `templates/{venue}.sty` or `templates/{venue}/`
   - <!-- bio-C10 --> Per-style content templates from `skills/paper-draft/references/templates/{paper_style}/` (when authored). Until they land: emit a 🟡-style warning in the terminal report that templates are not installed and proceed with the inline defaults below.
   - If no template: use generic article class; note in main.tex that the official template must be substituted
5. Generate `math_commands.tex`:
   - Collect all Formal notation from wiki/concepts/
   - Unify symbol definitions (vectors, matrices, sets, common operators)
6. Generate `main.tex` skeleton with style-dispatched section ordering:   <!-- bio-C10 -->

   **cs**:
   ```latex
   \documentclass{article}
   \input{math_commands}
   \usepackage{booktabs,graphicx,amsmath,hyperref,natbib}
   \title{<title>}
   \author{}
   \begin{document}
   \maketitle
   \begin{abstract}
   \end{abstract}
   \input{sections/introduction}
   \input{sections/related_work}
   \input{sections/method}
   \input{sections/experiments}
   \input{sections/conclusion}
   \bibliography{references}
   \bibliographystyle{plainnat}  % author-year
   \end{document}
   ```

   **bio**:   <!-- bio-C10 -->
   ```latex
   \documentclass{article}
   \input{math_commands}
   \usepackage{booktabs,graphicx,amsmath,hyperref}
   \usepackage[numbers,super,sort&compress]{natbib}
   \title{<title>}
   \author{}
   \begin{document}
   \maketitle
   \begin{abstract}
   \end{abstract}
   \input{sections/introduction}
   \input{sections/results}
   \input{sections/discussion}
   \input{sections/methods}
   \bibliography{references}
   \bibliographystyle{naturemag}  % Nature numeric, Vancouver
   \end{document}
   ```

   **clinical**:   <!-- bio-C10 -->
   ```latex
   \documentclass{article}
   \input{math_commands}
   \usepackage{booktabs,graphicx,amsmath,hyperref}
   \usepackage[numbers,sort&compress]{natbib}
   \title{<title>}
   \author{}
   \begin{document}
   \maketitle
   \begin{abstract}
   \end{abstract}
   \input{sections/introduction}
   \input{sections/methods}    % Methods front-loaded (clinical convention)
   \input{sections/results}
   \input{sections/discussion}
   \bibliography{references}
   \bibliographystyle{vancouver}  % NEJM/JAMA/Lancet
   \end{document}
   ```

### Step 2: Generate Figures and Tables

For each entry in the Figure Plan from PAPER_PLAN:

1. **Diagram type** (architecture diagrams, mechanism schematics):
   - cs: TikZ / pgfplots; matplotlib script if too complex
   - <!-- bio-C10 --> bio: BioRender-style mechanism schematics often originate from Inkscape / Adobe Illustrator and are imported as PDF; for fully programmatic diagrams use Inkscape SVG → PDF
   - <!-- bio-C10 --> clinical: CONSORT diagram conventions (boxes for enrolled / allocated / followed-up / analyzed); use TikZ or import SVG

2. **Plot type**:
   - cs: line plots (scaling), bar plots (ablation), scatter (qualitative); minimal captions
   - <!-- bio-C10 --> bio: multi-panel figures (a, b, c, ...) with **narrative captions** including:
     - sample sizes per panel
     - replicate types (biological vs technical, from `experiments.statistical_protocol == replicate_matrix_BxT`)
     - statistical test name + significance threshold (e.g. "n=4 biological replicates × 3 technical replicates; two-sided t-test, *P<0.05, **P<0.01")
     - error bar definition (SD vs SEM vs 95% CI)
   - <!-- bio-C10 --> clinical: forest plots (subgroup analysis), Kaplan-Meier curves (survival), funnel plots (meta-analysis); captions name the analysis population (intent-to-treat / per-protocol)

3. **Table type**:
   - Use booktabs style (toprule, midrule, bottomrule)
   - cs: best result bold, second-best underline
   - <!-- bio-C10 --> clinical: Table 1 is **always** baseline characteristics; columns = treatment groups + Total + P-value; rows = demographics + clinical covariates with appropriate descriptive statistics (mean ± SD or median [IQR] for continuous; n (%) for categorical)

### Step 3: Write Sections

For each section (in the **style-specific** outline order from PAPER_PLAN); if `--sections` is specified, write only those sections.

**3a. Collect Material** (style-dispatched)

From the section definition in PAPER_PLAN, extract claims, wiki page list, planned figures/tables, citation list. Read the relevant portions:

- **cs Introduction** → `wiki/ideas/{idea}.md#Motivation` + `wiki/topics/{topic}.md#Overview`
- **cs Related Work** → `wiki/papers/*.md#Related` + `wiki/concepts/*.md#Comparison`
- **cs Method** → `wiki/concepts/*.md#Formal_notation` + `wiki/claims/{claim}.md#Statement`
- **cs Experiments** → `wiki/experiments/*.md#Results` + `wiki/experiments/*.md#Analysis`
- **cs Conclusion** → `wiki/graph/open_questions.md` + `wiki/claims/*.md#Open_questions`

<!-- bio-C10 -->
- **bio Introduction** → wiki/ideas/{idea}.md#Motivation, but written in **biological prose** (no equations); pull a striking opening fact (often from `claims[*].evidence[*].type == clinical_validated`)
- **bio Results** → wiki/experiments/*.md#Results — one subsection per claim. For each `experiments[*].type`:
  - `validation` → headline result subsection
  - `mechanism` → "We tested whether the predicted mechanism holds" subsection (typically uses point-mutagenesis or chemical-probe data)
  - `dose_response` → titration subsection with EC50/IC50
  - `cross_context` → generalisation subsection
  - `negative_control` → folded into the validation subsection's caption ("a non-null negative control rules out X artefact")
- **bio Discussion** → start with the headline finding, then lit context, then mechanism interpretation, then explicit limitations (read from `wiki/graph/open_questions.md` and `wiki/claims/*.md#Conditions`)
- **bio Methods** → **wiki-driven serialization, not a writing task**:
  - Datasets subsection: serialize each `wiki/datasets/{slug}.md` with `versions[*].version`, `access`, `license`, `canonical_url` (A1)
  - Wet-lab assay subsections: read `wiki/experiments/{slug}.md setup.cell_line` (CVCL ID), antibody RRIDs from `setup` body, biological × technical replicate counts from `statistical_protocol == replicate_matrix_BxT`
  - Computational subsections: serialize `setup.weight_version`, `setup.force_field`, `setup.solvent_model`, `setup.simulation_length`, `setup.random_seed_protocol`
  - Statistical analysis subsection: serialize `statistical_protocol` value and parameters (1000 resamples for bootstrap_ci, k = min(5, n_positives) for stratified_kfold, etc.)
  - Resources subsection: serialize `estimated_cost.*` (A6) — gpu_hours, md_wallclock_hours, wet_lab_usd, fte_weeks, dataset_access_lead_time_days

<!-- bio-C10 -->
- **clinical Methods** → similar wiki-driven serialization, but front-loaded; explicitly cite the NCT ID and pre-registration source (B2 `clinical_trial_for.metadata.nct_id`)
- **clinical Results** → 3.1 participant flow (Figure 1 CONSORT), 3.2 baseline (Table 1), 3.3 primary outcome, 3.4 secondary outcomes, 3.5 safety
- **clinical Discussion** → principal findings → comparison with prior evidence → strengths and **limitations (mandatory, explicit)** → generalizability → conclusions

**3b. Write LaTeX**

Following `shared-references/academic-writing.md`:
- Write according to the paragraph plan for that section
- Insert `\cite{key}` citations (keys mapped from citation plan)
- <!-- bio-C10 --> citations are rendered per `\bibliographystyle`: cs uses `\citep{...}` author-year; bio/clinical use plain `\cite{...}` numeric Vancouver
- Insert `\ref{fig:name}` / `\ref{tab:name}` references to figures/tables
- Use symbols defined in `math_commands.tex`
- Begin each paragraph with a topic sentence
- cs Experiments: claim-first structure ("We claim X. To verify, we...")
- <!-- bio-C10 --> bio Results: result-first structure with embedded statistics ("In ABC cells, treatment with the phospho-PROTAC reduced target abundance by 78% (Fig. 2a; n=4 biological replicates, two-sided t-test, P<0.001)"). The figure reference, sample sizes, and statistics typically live in the same sentence.

**3c. De-AI Polish**

Same as legacy. <!-- bio-C10 --> Bio/clinical papers especially benefit from removing AI-style hedging — bio editors push back hard on "we delve into" / "this comprehensive analysis" prose.

**3d. Optional Review LLM Review (--review)**

If `--review` is enabled, for each section the persona is style-dispatched:

```
mcp__llm-review__chat:
  system: |
    {style-specific persona, see below}
  message: |
    ## Section: {section name}
    {LaTeX content}
    ## Paper style: {paper_style}                             <!-- bio-C10 -->
    ## Claims this section should support
    {claims from matrix}
    ## Review questions: {style-specific}
```

Personas:
- **cs**: "You are a senior ML researcher reviewing one section of a paper draft. Focus on: clarity, logical flow, claim-evidence alignment, notation consistency. Point out any remaining AI-sounding language patterns. Suggest specific rewrites for unclear passages."
- <!-- bio-C10 --> **bio**: "You are a senior bio researcher reviewing one section of a paper draft for {Nature / Cell / eLife}. For Results sections: confirm result-first structure, sample sizes in-line with claims, statistical test named in captions, biological vs technical replicate distinction maintained. For Methods: confirm cell line CVCL IDs, antibody RRIDs, dataset version pinning, replicate matrix declaration. For Discussion: confirm explicit limitations and biological mechanism interpretation."
- <!-- bio-C10 --> **clinical**: "You are a senior clinical reviewer for {NEJM / JAMA / Lancet}. For Methods: confirm NCT ID, pre-registration, primary endpoint definition, sample-size justification, pre-specified analyses. For Results: confirm CONSORT diagram, baseline table, ITT/per-protocol distinction, multiple comparisons handling. For Discussion: confirm explicit limitations (no hedging), generalizability statement."

Revise the section based on Review LLM feedback (inline edits; do not rewrite the entire section).

### Step 4: Build Bibliography

Following `shared-references/citation-verification.md`:

1. Collect all `\cite{key}` citations used across sections
2. For each citation, retrieve BibTeX from the PAPER_PLAN's citation plan:
   - Verified: write directly to `references.bib`
   - [UNCONFIRMED]: write to the bottom of `references.bib` with a `% [UNCONFIRMED]` comment
3. Exclude unused entries (do not use `\nocite{*}`)
4. Validate BibTeX format correctness (each entry has title, author, year)
5. <!-- bio-C10 --> For bio/clinical papers, also validate that PMID and/or DOI are present in the BibTeX entries — Vancouver style references typically display both. Missing both → 🟡 nudge.
6. Output bibliography statistics:
   ```
   references.bib: {N} entries, {M} verified, {K} [UNCONFIRMED]
   ```

### Step 5: Full-Paper Cross-Review

After all sections are complete, the cross-review system prompt is style-dispatched (same persona dispatch as Step 3d, but the questions are paper-level):

- **cs**: coherent story Intro→Conclusion, claim-evidence threading, notation consistency, figure/table references, redundancy.
- <!-- bio-C10 --> **bio**: Results-first storytelling holds up; mechanism is supported by ≥2 orthogonal perturbations; dose-response covers ≥3 orders of magnitude where applicable; cross-context retention threshold pre-registered; Methods is reproducible (CVCL, RRIDs, version pinning); Discussion limitations are explicit.
- <!-- bio-C10 --> **clinical**: pre-registration cited, primary endpoint analyzed in pre-registered way, CONSORT present, baseline table comprehensive, multiple comparisons handled, limitations explicit.

Make final adjustments based on Review LLM feedback.

### Step 6: Finalize Output

1. Confirm all files are written to the `paper/` directory
2. Verify basic integrity:
   - all files referenced by `\input{sections/X}` exist
   - all files referenced by `\includegraphics{figures/X}` exist
   - all `\cite{key}` keys have a corresponding entry in references.bib
   - all `\ref{label}` have a corresponding `\label{label}`
3. <!-- bio-C10 --> **Bio/clinical-specific integrity checks**:
   - Bio Methods serialization uses dataset versions that exist in the corresponding `wiki/datasets/{slug}.versions[]` (cross-reference C8 lint logic).
   - Clinical paper's NCT ID matches a B2 `clinical_trial_for` edge in `wiki/graph/edges.jsonl`.
4. Append log:
   ```bash
   python3 tools/research_wiki.py log wiki/ \
     "paper-draft | {paper_style} {venue} '{title}' | {N} sections, {M} figures, {K} citations ({V} verified)"
   ```
   <!-- bio-C10: log line carries paper_style for grep-ability -->
5. Print to terminal: same shape as legacy, plus `Paper style: {cs | bio | clinical}` line under `## Status`.

## Constraints

- **Each section draws from wiki**: do not generate content from nothing; every technical claim must trace back to a wiki page
- **BibTeX follows citation-verification.md**: fetch from DBLP/CrossRef/S2; do not generate from LLM memory
- <!-- bio-C10 --> **Bio/clinical BibTeX prefers CrossRef + PubMed over DBLP**: DBLP coverage of bio venues is sparse; running DBLP-first wastes time on misses.
- **De-AI polish is mandatory**: each section must receive a polish pass after writing; cannot be skipped
- **Figures follow academic-writing.md**: colorblind-safe, font size >= 8pt, vector format preferred
- <!-- bio-C10 --> **Bio figure captions include sample sizes + statistical test name**; clinical figure captions name the analysis population.
- **Anonymous submission**: do not write author names, institutions, or acknowledgements (per venue anonymity requirements)
- **\nocite{*} is forbidden**: only cite entries actually used
- **Notation consistency**: all sections use unified symbols from math_commands.tex
- **Back up existing paper/ before overwriting**: do not overwrite directly; back up first
- **Wikilink → \cite conversion**: [[slug]] references in PAPER_PLAN are converted to \cite{key} in LaTeX
- **Tables use booktabs**: no vertical lines or full grid
- <!-- bio-C10 --> **Bio Methods is a wiki-driven serialization, not a writing task**: pull cell lines (CVCL), antibodies (RRID), dataset versions, force fields, replicate counts, costs directly from `wiki/`. Do not paraphrase or invent these.
- <!-- bio-C10 --> **`paper_style` is read from PAPER_PLAN.md, not re-inferred**: if the metadata block is missing the field, error out and instruct the user to re-run /paper-plan.

## Error Handling

- **PAPER_PLAN not found**: error; suggest running /paper-plan first
- **PAPER_PLAN format incomplete**: list missing sections; suggest re-running /paper-plan
- <!-- bio-C10 --> **PAPER_PLAN missing `paper_style` metadata**: error with message "PAPER_PLAN was generated before C10 landed; rerun /paper-plan with --paper-style to regenerate, or manually add `paper_style: cs|bio|clinical` to the metadata block".
- **Wiki page not found** (claim/experiment/paper referenced in plan does not exist): warn and skip that reference; annotate as missing
- **Figure generation failed** (matplotlib error): output placeholder `% TODO: generate figure {name}`; continue with other sections
- **All BibTeX fetches failed**: use [UNCONFIRMED] placeholders; report the count requiring manual handling in the terminal
- **Review LLM unavailable** (--review mode): skip section review and cross-review; annotate as "unreviewed"
- **Venue template not found**: use generic article class; note in main.tex
- **Section too long** (exceeds plan page budget): warn; suggest moving to appendix or compressing
- <!-- bio-C10 --> **Bio Methods serialization fails because a wiki/datasets/{slug}.md does not exist**: emit a 🔴-style note in the terminal report and write a `% TODO: dataset entry missing — author wiki/datasets/{slug}.md` placeholder in the section. Do not block draft completion.
- <!-- bio-C10 --> **Clinical paper's NCT ID is missing from edges.jsonl**: emit a 🔴-style note; write a placeholder citation. The user must populate the B2 edge before submission.
