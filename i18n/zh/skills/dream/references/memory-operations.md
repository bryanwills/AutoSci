# /dream Memory Operations

`/dream` 是 SciMem 自演化 pass。它不是健康检查器，也不是内容生成器。它的工作是使记忆系统对自身可见地行动：弱痕迹淡化、分散痕迹变得有序、有用的潜在关联变为可审查的提案。其结果应改善未来的检索、构想和实验规划。

agent 应从已准备的上下文中做出语义记忆判断。工具的候选线索就像睡眠中的感觉输入：有用、不完整，且并非权威。

`/dream` 与策略运行时无关。语义判断可来自 Claude Code slash-command 会话、OpenAI 兼容 API 模型、本地模型，或另一个返回相同有据可查 JSON 契约的 agent。这并非更弱形式的记忆演化：自演化底层是将 SciMem 状态转化为已验证提案、受保护记忆变更和下游上下文变化的闭环。策略运行时可替换；证据、验证、来源可溯性和安全应用语义则不可替换。

一个提案只有在说明接受它将如何改变记忆未来行为时，才是 `/dream` 提案。若该操作仅修复 schema 问题、填充缺失字段或重复确定性评分，则它属于 `/dream` 之外。

当安全自动应用运行时，只有中/高置信度提案才可变为实际的记忆变更，且仅以可逆 frontmatter 元数据加 append-only 审计注释的形式进行。这使 `/dream` 拥有闭环，同时不允许其重写科学内容。已应用的元数据由下游上下文编译所使用：降权页面排名靠后，整合源折叠到其规范目标，关联出现在 SciEvolve 指导章节。

这一保守应用边界是无人值守记忆编辑 agent 的行业安全惯例，而非能力边界。相同的有据可查循环可通过安全应用自动运行，且显式 `--yolo` 或定时 `yolo=true` 可启用高置信度归档/合并路径。

## Forgetting

遗忘是一种可逆提案，用于降低弱、陈旧、嘈杂、已被取代或重复记忆痕迹的检索权重。

好的遗忘提案：

- 识别一个反复添加很少信号的页面、笔记、idea 或实验
- 解释为何该痕迹会污染未来的检索或推理
- 描述淡化它将如何使后续检索或规划更清晰
- 提出可逆操作，如归档、降权、标记为已取代、折叠到另一页面，或安排人工审查
- 引用来自页面内容、状态字段、先前失败 idea、信号或现有摘要的证据

安全应用效果：设置 `scievolve_memory_weight: downweighted` 并记录一条 `scievolve_dream_notes` 条目。不删除或归档页面。

弱遗忘提案：

- 仅"时间戳较旧"
- 仅"孤立页面"
- "缺失字段"或断开链接
- 任何自动删除
- 仅因 wiki 尚年轻而内容稀少的页面

有用遗忘理由示例：

- 一个失败的 idea 没有可复用的经验教训，却反复出现在活跃 idea 附近。
- 两条简短笔记重复了同一条已丢弃的实验路径；其中一条应折叠到失败记忆中。
- 一个概念被标记为已弃用，并已被具有更好证据的新概念页面取代。

## Consolidation

整合将分散的记忆痕迹组织成更强的单元。

在以下情况使用整合：wiki 中包含当前过于分散、检索无法很好利用的相关碎片：

- 近似重复的概念、方法、基础知识或主题碎片
- 共享机制、假设、数据集、失败模式或开放研究问题的多个页面
- 应成为可复用长期记忆的反复实验经验教训
- 相互关联但不完全相同、应聚类而非合并的页面

好的整合应使隐式记忆邻域变得显式。agent 应能说明碎片被分组后，哪个未来问题、搜索或实验计划将变得更容易。

安全应用效果：在目标页面的 `scievolve_consolidates_with` 中追加相关页面，并记录已应用的 dream id。

区分提案操作：

- `merge`：两个页面可能是同一概念/方法/主题
- `cluster`：多个页面应在主题或综合笔记下分组
- `summarize`：反复出现的实验或评审经验教训应成为可复用记忆
- `cross-link`：页面应保持独立，但应更易于相互跳转

避免不支持的整合：

- 不要仅因缺少 `parent_topics` 字段就推断主题归属
- 不要仅因标签重叠就合并
- 不要仅因共享关键词就合并不同的科学对象
- 不要在 `/dream` 期间重写页面内容

## Association

关联提出一个新的高价值关系，该关系从现有证据来看是合理的，但尚未明确存在。

好的关联提案：

- 连接现有的论文、概念、方法、主题、基础知识、idea 或实验
- 解释审查该链接的科学或工作流价值
- 解释该关联能解锁什么未来研究动作
- 引用具体证据：共享论文、共享实验、反复出现的机制、失败经验、信号或 graph 邻域
- 当连接是推测性而非已确立时，将置信度标记为低

关联是 `/dream` 最具 agentic 特色的部分：agent 应注意确定性扫描无法完全判断的潜在桥接。

有用的关联模式：

- method <-> concept：某个方法似乎实例化了某个概念，但关系缺失
- idea <-> method：一个活跃 idea 可能使用了已在长期记忆中的方法
- experiment <-> concept：某个结果揭示了关于更广泛机制的内容
- paper <-> topic：某篇论文支持一个正在形成的主题邻域
- foundation <-> method/concept：某个方法依赖于未声明的理论基础

不要提出捏造的链接。若证据有趣但薄弱，写一个低置信度待审查提案，而非将关联声称为事实。

安全应用效果：仅当置信度为中或高时，才将相关页面追加到 `scievolve_associations`。低置信度关联保持为提案状态。

## Choosing Proposal Count

优先选择 0-5 个有意义的提案。当记忆过于稀疏或没有可辩护的关联时，空操作的 dream 也是有效的。运行应体现判断，而非一份长长的确定性清单。
